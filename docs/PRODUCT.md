# Note 产品与架构取舍

本文只解释稳定的产品边界和实现理由。功能清单、安装和开发命令见根目录 `README.md`；历史视觉验收见 `design-qa.md`。

## 一个应用，两个同级模块

用户原来的有效动作是“每天写清单，然后一个个打勾”，所以 `Todo` 固定回到今天，避免先选分类、项目或视图。`Notes` 用于长期研究材料与结构化写作，不参与 Todo 的换日和顺延。两者可以互相链接，但完成、编辑、删除和换日不会暗改另一方内容。

## Todo：保持一页今日

- 前三项自动成为“今日三件”，其余进入“今天”，所有任务仍按用户手动顺序显示。
- 工作日到凌晨 04:00 才切换，避免午夜后的同一段研究工作被拆成两天。
- 未完成任务不会无限自动顺延；新工作日首次打开时逐项决定是否继续。
- 时间段是可选元数据：24 小时制、15 分钟步进、允许重叠和跨午夜，不参与排序。
- 主卡片只保留写、改、排、勾、撤销；低频能力进入托盘、快捷键或设置。

## Notes：笔记本优先但不做复杂知识库

- 层级固定为“笔记本 → 一级文件夹 → 笔记”，另有全部笔记、未分类和废纸篓。
- 不加入多级文件夹、标签、双向图谱或自动分类，避免桌面便笺演变成重型知识管理系统。
- 笔记可在未分类、笔记本根目录和不同一级文件夹之间移动；成功后继续打开原笔记。
- 删除先进入不会自动过期的废纸篓；正文与受管图片只在明确永久删除后清除。
- Markdown 导入和整库导出用于迁移与恢复，不把外部 Markdown 目录变成双向同步真源。

## 富文本与公式

可编辑真源是经过 allowlist 校验的 Tiptap/ProseMirror JSON。每次保存同时派生干净 Markdown 和纯文本，分别用于导出、旧数据兼容和搜索。字体、字号、下划线、行距等视觉元数据不会污染 Markdown。

字体按 Word 的字符系统思路处理：混合选区应用 Times New Roman 时，只修改英文、数字和西文符号；中文区段及其已有的宋体、楷体或黑体标记保持不变。

工具栏保留常见格式，低频格式收进弹层，并允许整体收起。粘贴只保留 Note 支持的结构；右键可选择纯文本粘贴。数学表达式保存为 `inlineMath`/`blockMath` 节点，由本地 KaTeX 渲染，失败时保留 LaTeX 源码而不清空正文。

表格与公式一样是可编辑的正式节点，不是渲染后不可修改的 HTML。工具栏使用网格选择尺寸，选中单元格后出现紧凑的上下文工具，可增删行列、切换首行表头、拖动列宽并删除整表；在末格按 `Tab` 自动新增一行。粘贴和 Markdown 导入识别规范 GFM 表格，旧笔记只迁移能被完整、无歧义识别的连续表格行。单表上限为 200 行 × 30 列，超过限制或包含不支持结构时保留原始文本，不能静默截断正文。导出统一生成可移植的 GFM 表格。

## 自适应窗口

Note 是无边框 Windows 窗口，最小 420×660，可从四边和四角缩放并标准最大化。420–639px 的 Notes 使用列表/编辑器单栏导航，640px 起使用双栏；最大化时阅读列仍保持有界居中。

默认桌面层不常驻最前，普通窗口自然覆盖 Note。快捷记录可临时来到前台，失焦后恢复原层级。Windows 拒绝特殊窗口层级时必须回退普通窗口并显示非阻断提示。任何点击或聚焦都不得缩放整张卡片。

## 单向数据流

```text
React renderer
  → preload 白名单 IPC
  → Electron main
  → shared/store.cjs 纯操作
  → 内存状态 + revision 广播
  → 临时文件原子替换 state.json
```

主进程是正式数据唯一真源。浏览器预览复用同一 Store，只另外负责 fixture、localStorage 和下载适配。共享能力同时供 CommonJS/ESM 使用时，只维护一个实现，另一种模块格式使用薄桥接层。

## 本地数据、升级与发布

正式数据和受管图片只位于当前 Windows 用户的 Electron `userData/note-data`。应用没有账户和远程同步端点，也不会把内容写入安装目录或 Git 仓库。

应用标识、产品名和数据目录跨版本保持稳定。升级前只保留最近一个兼容性快照；迁移失败自动回滚，旧版本拒绝覆盖新 schema 数据。正式 Release 由 GitHub Actions 从标签源码构建，只发布 NSIS 安装包和 SHA-256 校验文件。
